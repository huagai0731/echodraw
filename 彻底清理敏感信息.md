# 彻底清理 Git 历史中的敏感信息

## 方案 1：使用 BFG Repo-Cleaner（最简单，推荐）

### 步骤：

1. **下载 BFG**：
   - 访问：https://rtyley.github.io/bfg-repo-cleaner/
   - 下载 `bfg.jar` 文件

2. **执行清理**（在 Git Bash 中）：
```bash
# 1. 克隆一个裸仓库（用于清理）
cd ..
git clone --mirror https://github.com/huagai0731/echodraw.git echodraw-clean.git
cd echodraw-clean.git

# 2. 使用 BFG 删除 .env 文件
java -jar bfg.jar --delete-files .env

# 3. 清理和推送
git reflog expire --expire=now --all
git gc --prune=now --aggressive

# 4. 强制推送
git push --force

# 5. 回到原仓库，重新设置远程
cd ../echo
git remote set-url origin https://github.com/huagai0731/echodraw.git
git fetch origin
git reset --hard origin/master
```

## 方案 2：完全重置 master 分支（最彻底，但会丢失历史）

如果历史不重要，可以：

```bash
# 1. 备份当前代码
cd ..
cp -r echo echo-backup

# 2. 删除 .git 目录
cd echo
rm -rf .git

# 3. 重新初始化 Git
git init
git add .
git commit -m "Initial commit (cleaned)"

# 4. 强制推送到远程
git remote add origin https://github.com/huagai0731/echodraw.git
git push -u origin master --force
```

## 方案 3：使用 git-filter-repo（现代工具）

```bash
# 1. 安装 git-filter-repo
pip install git-filter-repo

# 2. 从历史中移除 .env
git filter-repo --path backend/.env --invert-paths

# 3. 强制推送
git push origin --force --all
```

## 方案 4：只清理特定提交（如果知道是哪个提交）

```bash
# 1. 查看提交历史，找到包含 .env 的提交
git log --all --full-history --oneline -- backend/.env

# 2. 假设提交 hash 是 956aaac，使用交互式 rebase
git rebase -i 956aaac^1

# 在编辑器中，将包含 .env 的提交标记为 'edit'
# 然后：
git rm --cached backend/.env
git commit --amend --no-edit
git rebase --continue

# 3. 强制推送
git push origin --force master
```

## 如果还是不行，检查是否有其他文件

```bash
# 搜索所有可能包含敏感信息的文件
git log --all --full-history -S "TOS_ACCESS_KEY_ID" --source --all
git log --all --full-history -S "VolcEngine" --source --all
```
